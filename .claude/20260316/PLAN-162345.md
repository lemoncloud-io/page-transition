## API Design

### Configuration-based Platform Detection

```typescript
// Types
interface PageTransitionConfig {
    /** Platform for animation style. 'auto' uses navigator.userAgent */
    platform?: 'ios' | 'android' | 'auto';
    /** Custom platform detector function */
    detectPlatform?: () => 'ios' | 'android';
}

// Provider (optional)
<PageTransitionProvider config={{ platform: 'auto' }}>
    <App />
</PageTransitionProvider>

// Or per-hook
const navigate = useNavigateWithTransition({ platform: 'ios' });
```

### Exports

```typescript
// Hooks
export { useNavigateWithTransition } from './hooks/useNavigateWithTransition';
export { useGoBack } from './hooks/useGoBack';

// Types
export type {
    TransitionNavigateOptions,
    NavigateWithTransitionFn,
    PageTransitionConfig,
    PlatformType
} from './types';

// Constants
export { BACK_NAVIGATION_CLASS, ANDROID_PLATFORM_CLASS } from './constants';

// Utils
export { detectPlatform } from './utils/platform';
```

## Decisions

| Choice | Options | Selected | Rationale |
|--------|---------|----------|-----------|
| Build tool | tsup, tsdown, unbuild | tsup | Most stable, well-documented |
| Platform detection | Hardcoded, configurable | Configurable | More flexible for different apps |
| Provider pattern | Required, optional | Optional | Simpler API, config per-hook also works |
| CSS delivery | Bundled, separate file | Separate file | Tree-shaking friendly, standard pattern |
| npm publish | NPM_TOKEN, Trusted Publishing | Trusted Publishing | More secure, no token management |
